# Finance CRM

Finance CRM - це внутрішня CRM-система для страхової компанії, призначена для автоматизації роботи з клієнтами, полісами, оплатами та страховими випадками.

Система орієнтована на щоденну роботу співробітників компанії: менеджерів, керівників відділу та адміністраторів. Основний інтерфейс реалізований у вигляді адміністративної панелі на Filament.

## Основні можливості

- ведення клієнтської бази;
- оформлення та супровід страхових полісів;
- облік оплат за полісами;
- реєстрація страхових випадків;
- ведення внутрішніх нотаток до заявок;
- розмежування доступу за ролями користувачів;
- аналітика та звітність для керівника;
- контроль строків дії полісів.

## Предметна область

Система моделює внутрішні процеси страхової компанії, у межах яких:

- менеджери працюють із закріпленими за ними клієнтами;
- для клієнтів підбираються страхові пропозиції;
- на основі пропозицій оформлюються поліси;
- фіксуються оплати та статуси договорів;
- у разі настання страхового випадку створюється відповідна заявка;
- керівник відділу контролює навантаження менеджерів та ключові показники роботи.

## Основні сутності

### Клієнти
У системі зберігається інформація про клієнтів:
- фізична особа або компанія;
- контактні дані;
- джерело звернення;
- закріплений менеджер;
- статус клієнта.

### Поліси
Для кожного полісу зберігаються:
- номер полісу;
- клієнт;
- страхова пропозиція;
- менеджер;
- дата початку та завершення дії;
- сума оплати;
- сума покриття;
- статус полісу.

### Оплати
Для полісів ведеться облік оплат:
- спосіб оплати;
- статус оплати;
- строк оплати;
- сума;
- службова інформація про транзакцію.

### Страхові випадки
Система підтримує реєстрацію страхових випадків:
- номер заявки;
- пов’язаний поліс;
- хто повідомив про випадок;
- статус;
- сума вимоги;
- резерв / виплата;
- опис та додаткові дані.

### Нотатки до заявок
Співробітники можуть залишати внутрішні нотатки до страхових заявок для супроводу та фіксації деталей.

## Ролі користувачів

У системі передбачені такі ролі:

- **admin** - повний доступ до системи;
- **supervisor** - керівник відділу;
- **manager** - менеджер.

Базова логіка доступу:
- менеджер працює лише зі своїми клієнтами та полісами;
- керівник бачить загальну аналітику та результати роботи всіх менеджерів;
- адміністратор керує системою повністю.

## Поточний функціонал

На даний момент у проєкті реалізовано:

- базові CRUD-операції для ключових сутностей;
- створення та ведення клієнтів;
- оформлення полісів;
- створення записів про оплату;
- автоматичну генерацію номера полісу та номера заявки;
- базову логіку зміни статусів полісів та оплат;
- адміністративну панель для внутрішньої роботи співробітників.

## Подальший розвиток

Система може бути розширена такими можливостями:

- внутрішні сповіщення про завершення строку дії полісів;
- дашборд з основною статистикою;
- звіти по роботі менеджерів;
- журнал активності користувачів;
- публічна форма заявки для нових клієнтів;
- експорт звітів у файл;
- розширені механізми фільтрації та пошуку.

## Приклад бізнес-процесу

1. Менеджер створює нового клієнта або отримує вхідну заявку.
2. Для клієнта підбирається страхова пропозиція.
3. На основі пропозиції оформлюється поліс.
4. У системі фіксується оплата та поточний статус полісу.
5. У разі настання страхового випадку створюється заявка.
6. Керівник переглядає аналітику, навантаження менеджерів та ключові показники через дашборд і звіти.

## Встановлення та запуск

### 1. Клонування репозиторію
```bash
git clone https://github.com/meowam/finance-crm.git
cd finance-crm


2. Встановлення PHP-залежностей
composer install


3. Встановлення frontend-залежностей
npm install


4. Підготовка середовища
cp .env.example .env
php artisan key:generate


5. Налаштування бази даних

За замовчуванням можна використовувати SQLite:

touch database/database.sqlite
php artisan migrate

За потреби можна налаштувати MySQL або іншу підтримувану СУБД через .env.

6. Запуск проєкту
composer run dev

Або окремо:

php artisan serve
npm run dev

Вхід до системи

Адміністративна панель доступна за адресою:

/admin

Для входу потрібен активний користувач із роллю admin, supervisor або manager.

Структура системи

Основні модулі проєкту:

Clients - робота з клієнтами;
Policies - оформлення та супровід полісів;
Policy Payments - облік оплат;
Claims - страхові випадки;
Claim Notes - внутрішні нотатки до заявок;
Users - користувачі та ролі.
